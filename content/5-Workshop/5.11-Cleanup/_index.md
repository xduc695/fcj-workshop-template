To avoid incurring unexpected charges on your AWS account after completing the lab, please proceed to clean up and delete all initialized resources in the exact order instructed below.

---

### Step 11.1: Delete ECS Services & Clusters
1. Go to the **ECS** service -> click the **`pg-cluster`** Cluster.
2. Select the **Services** tab:
   - Check the **`pg-frontend-service`** service -> Click the **Delete** button -> Enter the confirmation text to delete.
   - Check the **`pg-backend-service`** service -> Click the **Delete** button -> Confirm deletion.
3. Wait for the **Tasks** to stop completely.
4. Click the **Delete cluster** button in the upper right corner to delete the Cluster.

---

### Step 11.2: Delete Application Load Balancer & Target Groups
1. Go to the **EC2** service -> scroll down to the **Load Balancing** section:
   - Click **Load Balancers** -> Check **`pg-alb`** -> Click **Actions** -> select **Delete load balancer** -> Confirm deletion.
   - Click **Target Groups** in the left menu -> Check both Target Groups **`tg-frontend`** and **`tg-backend`** -> Click **Actions** -> select **Delete** -> Confirm deletion.

---

### Step 11.3: Delete RDS PostgreSQL Database
1. Go to the **RDS** service -> select **Databases** in the left menu.
2. Check your **`pg-db`** database.
3. Click the **Actions** button -> select **Delete**.
4. Configure the deletion page:
   - **Uncheck** *"Create final snapshot?"* (Do not take a final snapshot to avoid storage charges).
   - **Check** *"I acknowledge that..."* to confirm.
   - Enter the confirmation keyword (usually `delete me`) in the empty box.
5. Click the **Delete** button and wait for the database deletion to complete (usually takes 5-10 minutes).
6. Click the **Subnet groups** section in the left RDS menu -> Select and delete **`pg-db-subnet-group`**.

---

### Step 11.4: Delete NAT Gateway & Release Elastic IP
The NAT Gateway is a resource with a very high hourly charge, so it needs to be deleted immediately:
1. Go to the **VPC** service -> select **NAT gateways** in the left menu.
2. Check the NAT Gateway **`pg-nat`** (or the NAT generated by the VPC Wizard).
3. Click **Actions** -> select **Delete NAT gateway** -> Confirm deletion.
4. **Release Elastic IP:**
   - Select the **Elastic IPs** section in the left VPC menu.
   - Check the static IP previously associated with the NAT gateway.
   - Click **Actions** -> select **Release Elastic IP addresses** -> Confirm release. *(If you do not release unused static IPs, AWS will still charge your account)*.

---

### Step 11.5: Delete VPC
1. In the **VPC** service, select **Your VPCs** in the left menu.
2. Check your **`pg-vpc`** VPC.
3. Click **Actions** -> select **Delete VPC**.
4. AWS will automatically analyze and clean up all Subnets, Internet Gateways, Route Tables, and security groups associated with this VPC. Click **Delete VPC** to confirm.

---

### Step 11.6: Delete ECR Repositories, S3 Bucket, and KMS Key
1. **ECR Repositories:** Go to the **ECR** service -> Check the 2 repositories `pg-combined-backend` and `pg-frontend` -> Click **Delete** to delete the Docker Images.
2. **S3 Bucket:**
   - Go to the **S3** service -> Click the S3 Bucket **`pg-db-backups-795938553282-ap-southeast-1-an`**.
   - Click the **Empty** button to delete all snapshot files inside -> Confirm.
   - Go back to the Bucket list -> Check that bucket and click the **Delete** button -> Enter the bucket name to confirm permanent deletion.
3. **KMS Key:** Go to the **KMS** service -> click the **`pg-s3-export-key`** key -> click **Key actions** -> select **Schedule key deletion** -> Select the minimum number of days (7 days) -> click confirm.

Congratulations on successfully and securely completing the lab!
