Event 1

Summary Report “FCJ Community Day - AI & Prompt Engineering”

Event Objectives

  • Share foundational knowledge about AI, Cloud, Prompt Engineering, and how to apply AI in learning, work, and software development.
  • Help attendees understand how AI is changing learning mindsets, professional skills, and product development processes.
  • Guide effective communication with AI through clear, structured prompts that save token costs.
  • Provide directions for personal development, professional attitude, and adaptability in the AI era.
  • Introduce the use of AI Agents to support the software development lifecycle according to Software Engineering standards.

Speaker List

  • Huynh Hoang Long - Shared how social networks use psychological mechanisms like dopamine, instant feedback, and FOMO (fear of missing out) to maintain attention, and suggested how to apply this to learning.
  • Nguyen Tuan Thinh - Presented the topic “Automated Prompt Engineering: Enhancing LLM Output Quality”, focusing on prompt optimization techniques, Token Economics, the Proptimizer tool, and AI applications on AWS.
  • Khang - Shared about personal development mindsets, how AI is changing humans, growth mindset, professional attitude, integrity, and how companies evaluate personnel.
  • Thảo - Presented the topic “Don’t just Vibe Code – Build Software Engineering Standards”, focusing on using AI Agents and the BMAD Method in a professional software development process.

Key Highlights

Applying Social Network Mechanisms to Learning

  • Attention Mechanism: Analyzed why the brain is often more attracted to social media than studying.
  • Key Psychological Factors:
    • Dopamine: Creates a sense of reward and anticipation.
    • Instant Feedback: Helps users see results immediately.
    • Short-term Rewards: Maintains continuous returning behavior.
    • Fear of Missing Out (FOMO): Makes it hard for users to leave the platform.

From this, attendees understood that learning can be designed to be more engaging by breaking down goals, creating quick feedback, and helping learners feel continuous progress.

Prompt Engineering and Optimizing AI Output Quality

  • Common Problem: Generic prompts often generate generic, superficial results that are hard to apply in real work.
  • Structure of a Good Prompt:
    • Role
    • Task
    • Context
    • Input Data
    • Examples
    • Output Format
    • Specific Constraints

The presentation also covered Token Economics, showing that optimizing prompts not only yields better results but also saves costs when using AI. Additionally, the Proptimizer tool was introduced to help users optimize prompts and communicate with AI more effectively.

AI and AWS Applications in Product Development

  • AI Deployment Architecture: Introduced technical components for deploying AI applications on the AWS platform.
  • Highlighted Services:
    • API Gateway
    • AWS Lambda
    • Amazon Cognito
    • Amazon Bedrock
    • Amazon DynamoDB
    • CloudWatch

This gave attendees a comprehensive view of how an AI system is built from backend, security, data storage, AI model integration, to operational monitoring.

Growth Mindset and Career Orientation in the AI Era

  • AI Amplifies Personal Capabilities: AI can help people work faster, but effectiveness depends on how users think and coordinate the tools.
  • Key Factors in Personal Development:
    • Learning Attitude
    • Critical Thinking
    • Self-Direction Ability
    • Integrity at Work

Topics like growth mindset, asking “Why?”, and the 3-circle model in work helped attendees clearly see the requirements for personal development in a tech environment.

Software Development with AI Agents and Software Engineering Standards

  • Core Message: Do not just “vibe code” (using AI to write code based on feelings), but build a standard Software Engineering process.
  • Introduced AI Agents:
    • PM Agent: Supports creating PRDs.
    • Architect Agent: Supports architectural design.
    • Scrum Master: Divides Epics/Stories and manages progress.
    • Developer Agent: Writes code for each Story.
    • Review Agent: Tests, refactors, and reviews the codebase.

This content helped attendees understand that AI is not just for fast coding, but can support the entire software development process if coordinated correctly.

What I Learned

Learning Mindset and Using AI

  • Do not use AI passively: Users need to know how to ask clear questions, provide appropriate context, and verify the output.
  • Design learning more effectively: Apply quick feedback mechanisms, small goals, and a sense of continuous progress to maintain learning motivation.
  • Prompt Engineering is a crucial skill: A good prompt helps AI generate accurate, clear results that better fit practical needs.

Professional Mindset and Personal Development

  • Growth Mindset: Proactively ask “Why?” to understand the root of the problem instead of just following instructions.
  • Integrity at Work: Doing well even when unsupervised is an important factor in a professional environment.
  • Humans remain at the center: AI can provide strong support, but humans still play the role of coordinating, verifying, and taking final responsibility.

Technical Architecture and Software Development

  • Understand the AI application deployment process on AWS: From APIs, authentication, data storage, AI model integration, to operational monitoring.
  • Applying AI Agents in software processes: AI can assist from requirements analysis, architectural design, coding, testing, to reviewing.
  • Software Engineering Mindset: Software development with AI requires a clear process, quality control, and long-term maintainability.

Application to Work

  • Optimize AI usage: Apply a prompt structure containing role, task, context, input data, examples, and constraints to get better results.
  • Improve learning habits: Break down learning goals, create quick feedback, and track progress to maintain motivation.
  • Apply AI in software development: Try using AI Agents to support PRD creation, task division, coding, reviewing, and testing.
  • Hone professional mindset: Proactively ask questions, verify results, and take responsibility for the products created.
  • Learn more about AWS: Research services like Lambda, Bedrock, DynamoDB, Cognito, and CloudWatch to better understand how real-world AI systems are deployed.

Event Experience

Attending the FCJ Community Day - AI & Prompt Engineering was a useful experience, helping me approach AI from various perspectives: learning, prompt engineering, career orientation, and software development. Some highlighted experiences:

Approaching AI from a learning perspective

  • The sharing session on social media mechanisms helped me understand why learning is often harder to maintain than short-term entertainment activities.
  • I realized that if I know how to design small goals, create quick feedback, and maintain a sense of progress, learning can become more engaging and sustainable.

Learning effective communication with AI

  • The content on Prompt Engineering helped me understand that AI’s output quality heavily depends on the input quality.
  • Instead of asking short and generic questions, I need to clearly define the role, context, goal, format, and limits so AI can respond more accurately to my needs.

A clearer view of the human role in the AI era

  • The sharing on growth mindset and integrity helped me understand that technology is merely a supporting tool, while learning attitude and personal responsibility remain the key factors.
  • AI can speed up work, but humans still need critical thinking to verify and coordinate the results.

Visualizing the software development process with AI

  • The presentation on AI Agents showed me how AI can participate in multiple steps of the software development process.
  • I understand that using AI in programming shouldn’t stop at generating code quickly, but must be tied to standard processes, testing, reviewing, and quality control.

Lessons Learned

  • AI is a powerful supporting tool, but it does not completely replace human thinking and responsibility.
  • Prompt Engineering is an essential skill for using AI effectively in learning and work.
  • Learning can be improved by breaking down goals, creating quick feedback, and maintaining motivation.
  • In a tech environment, attitude, growth mindset, and integrity are as important as technical skills.
  • Software development with AI needs a clear process; one should not solely rely on auto-generated code.
